# output

`output` モジュールは wikiextractor のディレクトリ命名規則に従い、抽出された記事を分割出力ファイルに書き込みます。

## 型

### `OutputConfig`

出力スプリッターの設定。

| フィールド | 型 | 説明 |
| ----- | ----- | ----- |
| `path` | `PathBuf` | 出力ディレクトリパス、または `"-"` で stdout |
| `max_file_size` | `u64` | 出力ファイルあたりの最大バイト数 |
| `compress` | `bool` | bzip2 で出力を圧縮するかどうか |

### `OutputSplitter`

ファイルローテーションと書き込みを管理します。必要に応じてサブディレクトリとファイルを作成します。

## ディレクトリ命名規則

出力ファイルは wikiextractor の命名規則で整理されます:

```text
output/
  AA/
    wiki_00
    wiki_01
    ...
    wiki_99
  AB/
    wiki_00
    ...
```

- 各ディレクトリには最大100ファイルが格納
- ディレクトリ名は AA、AB、...、AZ、BA、...、ZZ のパターン
- `compress` 有効時はファイル名が `wiki_00.bz2` のようになる

## 特殊モード

- **stdout モード** -- `path` が `"-"` の場合、分割せずにすべての出力を stdout に書き込み
- **ゼロサイズ** -- `max_file_size` が 0 の場合、各記事を個別のファイルに書き込み
